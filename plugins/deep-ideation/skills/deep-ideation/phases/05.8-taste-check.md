# Phase 5.8: TASTE CHECK (Orchestrator Gate)

This phase is NOT a subagent. The orchestrator runs it directly using `AskUserQuestion`. No new agent is spawned — no extra tokens.

## Who Runs This

The orchestrator itself. Reads the idea DB, selects top 10 by diversity, presents to user.

**Skip conditions:**
- `--no-checkpoints` flag → silently skip, log to workspace
- LITE mode → show the gate but include "Skip" as an option
- STANDARD / DEEP → checkpoint is required (skip option not shown)

## Step 1: Select Top 10 by Diversity

```bash
# Get all transform-phase ideas with zone and tag info
python scripts/idea_db.py show <workspace> --columns "id,name,description,temperature_zone,tag,source_agent"
```

From this list, pick 10 ideas that span:
- At least 3 different temperature zones (FIRE, PLASMA, ICE, GHOST, CHAOS)
- A mix of tags (at least one BOLD/WILD and one SAFE)
- Different source agents (no single John dominates the shortlist)

If fewer than 10 transform-phase ideas exist (LITE mode), show all.

## Step 2: Present to User

```
AskUserQuestion:
  question: "Before synthesis — which of these directions resonate with you?

    This is a taste check, not a hard filter. Ideas you pick get a
    small boost in Synthesizer ranking; others stay in the pool.

    [1] #ID Name — Description (zone: FIRE)
    [2] #ID Name — Description (zone: ICE)
    [3] #ID Name — Description (zone: PLASMA)
    ... (up to 10)

    Type the numbers of any that feel promising, or choose an option below."
  header: "Taste Check (Phase 5.8)"
  options:
    - "Type numbers, e.g. 1, 3, 7"
    - "None stand out — continue without favorites"
    - "Skip this checkpoint"   ← shown in LITE mode only
```

Parse the user's text response to extract idea IDs. If the user picks option 2 ("None stand out"), proceed with no favorites recorded.

## Step 3: Record Favorites in DB

```bash
# Ensure column exists (idempotent)
python scripts/idea_db.py add_column <workspace> user_favorites --default ""

# Mark each selected idea
python scripts/idea_db.py mark_favorites <workspace> --ids "1,3,7"
# Sets user_favorites="true" for each given ID
```

Log the result to `$WORKSPACE/05.8-taste-check.md`:
```
## Taste Check Results
- Presented: #ID1, #ID2, ... #ID10
- User favorites: #ID1, #ID3, #ID7
- Checkpoint: completed / skipped / no favorites
```

## How Favorites Are Used Downstream

- **Phase 8b (HYBRIDIZE):** Synthesizer reads `user_favorites` column. Favorites get +10% weight boost when ranking convergent signals. Not a hard override — a low-signal favorite can still rank behind a high-signal non-favorite.
- **Phase 9 (CONVERGE):** Converge notes favorites in its final recommendation rationale. If a favorite ranks in top 3, it is highlighted.

```bash
# Downstream agents read favorites via:
python scripts/idea_db.py filter <workspace> user_favorites true
```

## Anti-Patterns

- **Don't run as a subagent** — the whole point is zero extra token cost for asking the user
- **Don't hard-filter** — non-favorites stay in the pool; favorites only get a bounded boost
- **Don't skip in STANDARD/DEEP** unless `--no-checkpoints` is set
