## The Six Hats

Apply each hat to the top 10 ideas from Phase 6. Not every hat needs a paragraph — a bullet is sufficient. The goal is breadth, not depth.

### Selecting the Top 10

```bash
# Get the top 10 ideas from the build phase
python scripts/idea_db.py filter <workspace> phase build
# If fewer than 10 build-phase ideas, also include top transform-phase ideas:
python scripts/idea_db.py filter <workspace> phase transform
```

Use the IDs from this output for all hat evaluation and subsequent `set` commands.

### White Hat — Facts & Evidence
For each idea: What data do we have? What's unknown? What assumptions are we making?
- Focus on: data that supports, data that contradicts, unknown assumptions
- Output: `WHITE: supports=[facts], missing=[gaps], assumes=[assumptions]`

### Red Hat — Gut / Intuition
For each idea: How does this feel? Exciting? Scary? Boring? Energizing?
- No justification — pure instinct
- Output: `RED: feels [emotion] — [one phrase why]`

### Black Hat — Critical Risk
For each idea: What could go wrong? Hidden costs? Unintended consequences?
- DO NOT use Black Hat to kill ideas — use it to find Invert targets
- Output: `BLACK: risk=[risk], invert candidate: [yes/no + what to flip]`

### Yellow Hat — Upside Potential
For each idea: Best case scenario? Who benefits most? What's the ceiling?
- Output: `YELLOW: best case=[scenario], beneficiary=[who], ceiling=[value estimate]`

### Green Hat — Alternatives & Variations
For each idea: What variations exist? What if we combined this with [other idea]?
- Green Hat is a generation trigger — it often produces new seed candidates
- Output: `GREEN: variations=[list], combine with=[idea #]`

### Blue Hat — Process Fit
For each idea: Does this fit the current constraints and context?
- Focus on: implementation path, team fit, timeline realism
- Output: `BLUE: process fit=[smooth/friction/blocker], needed change=[what]`

## Writing to the Idea Database

```bash
# Discover current schema
python scripts/idea_db.py describe <workspace>

# If Green Hat surfaced new seeds, add them as new rows
# The output will print IDs: 65,66... — pass to orchestrator for downstream use
python scripts/idea_db.py add_batch <workspace> green-hat-seeds.json
# JSON format: [{"name":"...","description":"...","source_agent":"Hat Eval (Green Hat)","chain":"Green Hat variation of #12","tag":"BOLD","phase":"hat_eval"}]

# Record hat evaluation columns on the top 10 evaluated ideas
python scripts/idea_db.py add_column <workspace> hat_red --default ""
python scripts/idea_db.py add_column <workspace> hat_black_invert --default "no"

# Set per idea using its existing ID
python scripts/idea_db.py set <workspace> <id> hat_red "exciting"
python scripts/idea_db.py set <workspace> <id> hat_black_invert "yes"
```

## Notes for Tension Analyzer

After Phase 6.5, the Tension Analyzer receives:
- All Phase 6 ideas with chains
- The Hat Evaluation Pass output
- Flagged invert candidates (from Black Hat)
- Suggested cross-combinations (from Green Hat)
- Gut-check ranking (from Red Hat)

The Tension Analyzer should prioritize tensions between:
- High Red Hat excitement + High Black Hat risk (emotionally appealing but risky)
- Different Red Hat gut reactions across temperature zones (FIRE loves it, ICE hates it)
- Green Hat combinations that create a new bridge idea
