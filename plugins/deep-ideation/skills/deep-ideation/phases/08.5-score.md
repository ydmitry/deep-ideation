# Phase 8.5: SCORE

Launch the Scorer. See `agents/scorer.md`.

## When to Run

After Phase 8 (SYNTHESIZE) and before Phase 9.5 (STRESS-TEST). Runs in ALL modes.

The Synthesizer defines criteria and weights. The Scorer applies them. This separation prevents the idea-generating agent from scoring its own output.

## What It Receives

- `$WORKSPACE/08-synthesize.md` — the Synthesizer's criteria list, weights, and rationale
- `$WORKSPACE/ideas.csv` — every idea from the session
- `$WORKSPACE/01-discover.md` — root causes (for context on what criteria mean)
- `$WORKSPACE/07-tension.md` (if it exists) — session tensions

## What It Produces

1. **Updated `ideas.csv`** — new/filled columns:
   - criteria columns (filled by the Scorer, e.g., `feasibility`, `novelty`, and session-specific criteria)
   - `total_score` (weighted average using Synthesizer's weights)
   - `composite_score` (seeded from `total_score`; updated by Stress-Tester and Brilliance later)
   - `z_score` (per-idea Z-score relative to the scored cohort)
   - `evidence_ref` (per-idea citation of upstream sources that justify the scores)
   - `menu_bucket` (qualitative label: `quick_win`, `core_bet`, `moonshot`, or empty)
   - `stress_multiplier` (default 1.0; mutated by Stress-Tester in Phase 9.5)
   - `brilliance_multiplier` (default 1.0; mutated by Brilliance in Phase 10)
   - `score_notes` (running log; appended by Stress-Tester and Brilliance)

2. **`$WORKSPACE/scorer_drop_log.md`** — ideas from the expanded cohort excluded from scoring, with reason codes

3. **Ranked output** saved to `$WORKSPACE/08.5-score.md`:
   - Session stats (cohort size, excluded count)
   - Full ranked table (including z_score column)
   - Idea Menu with qualitative bucket assignments
   - Notes on unscored ideas

## What It Does NOT Do

- Does NOT generate new ideas
- Does NOT redefine criteria or weights (those belong to the Synthesizer)
- Does NOT invent business precision (market size, effort days, revenue)
- Does NOT overwrite `stress_multiplier` or `brilliance_multiplier` (those belong to later phases)

## Key Commands

```bash
# Discover current schema — confirm criteria columns exist from Synthesizer
python scripts/idea_db.py describe <workspace>

# Ensure composite pipeline columns exist
python scripts/idea_db.py add_column <workspace> stress_multiplier --default "1.0"
python scripts/idea_db.py add_column <workspace> brilliance_multiplier --default "1.0"
python scripts/idea_db.py add_column <workspace> composite_score --default ""
python scripts/idea_db.py add_column <workspace> z_score --default ""
python scripts/idea_db.py add_column <workspace> score_notes --default ""
python scripts/idea_db.py add_column <workspace> evidence_ref --default ""

# Check cohort size — in STANDARD mode, abort if N < 30
python scripts/idea_db.py size <workspace>

# Write scorer_drop_log.md for any excluded ideas
python scripts/idea_db.py scorer_drop_log <workspace> drops.json

# Validate evidence refs BEFORE writing scores (exits non-zero if refs are missing)
python scripts/idea_db.py validate_evidence_refs <workspace> scores.json

# Write per-idea scores (including evidence_ref field)
python scripts/idea_db.py set_batch <workspace> scores.json

# Compute total_score via weighted average (Synthesizer's weights)
python scripts/idea_db.py compute <workspace> \
  --criteria "feasibility,novelty,[session-criteria]" \
  --target "total_score" \
  --formula weighted_avg \
  --weights "feasibility:2,novelty:3,[root_cause_crit]:4,[tension_crit]:2"

# Seed composite_score from total_score (multipliers default to 1.0)
python scripts/idea_db.py compute_composite <workspace>

# Compute Z-scores across the full scored cohort
python scripts/idea_db.py compute_zscores <workspace> --source composite_score --target z_score

# Add menu_bucket and assign buckets to top candidates
python scripts/idea_db.py add_column <workspace> menu_bucket --default ""
python scripts/idea_db.py set <workspace> <id> menu_bucket "quick_win"
python scripts/idea_db.py set <workspace> <id> menu_bucket "core_bet"
python scripts/idea_db.py set <workspace> <id> menu_bucket "moonshot"

# Export ranked table for the output file (include z_score)
python scripts/idea_db.py export_md <workspace> \
  --columns "id,name,source_agent,temperature_zone,total_score,z_score,menu_bucket" \
  --sort total_score --desc
```

## Forced Distribution

The Scorer must place at least one idea at score **1** and at least one at score **5** on each criterion across the full cohort. Without this constraint, all scores cluster in a narrow window and ranking becomes decorative.

## Evidence Discipline

Every score requires an `evidence_ref` citing a specific upstream idea-ID or artifact snippet. Run `validate_evidence_refs` before `set_batch`. Missing refs block the write — an unattributed score is a fabricated score.

## Expected Output

Save to `$WORKSPACE/08.5-score.md`. This is the authoritative ranking for downstream phases. The Brilliance Filter appends to this file in Phase 10.

## Transition

After Scorer completes:
- **STANDARD/DEEP:** proceed to Phase 9.5 STRESS-TEST (reads `top <workspace> composite_score --n 5`)
- **LITE:** proceed directly to Phase 10 BRILLIANCE FILTER

## Anti-Patterns

- **Don't score only hybrids.** The full cohort is: seed, transform, ratchet, bridge, hybrid, build, green_hat. Silently dropping upstream phases is what broke the old approach.
- **Don't let the Scorer redefine criteria.** The Synthesizer owns criteria selection; the Scorer owns application.
- **Don't bucket every idea.** Most ideas should have `menu_bucket` empty. Aim for 3-5 per bucket, maximum.
- **Don't skip forced distribution.** A cohort where every idea scores 3-4 on every criterion is an uninformative cohort.
- **Don't skip evidence refs.** If you can't cite an upstream source, you're scoring from memory or invention, not from the session's actual output.
