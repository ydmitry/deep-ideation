# Phase 5.7: DIALECTICAL RATCHET

Launch the Dialectical Ratchet agent. See `agents/dialectical-ratchet.md`.

**Runs in:** STANDARD and DEEP modes only. Skip in LITE.

## When It Runs
After Phase 5.5 (COLLISION MAP) — hot zones must be identified first.
Before Phase 6 (BUILD) — syntheses feed into the Brainwriter as pre-resolved ideas.

## What It Processes
Only HOT zones from the Collision Map. WARM zones go to the Tension Analyzer (Phase 7).

## What It Receives
- Hot zone details from `$WORKSPACE/05.5-collision-map.md`
- TRIZ Contradiction Card from `$WORKSPACE/01-discover.md`

## Mode Behavior
| Mode | Hot Zones | Cycles per Zone |
|------|-----------|----------------|
| LITE | Skip | Skip |
| STANDARD | Max 2 zones | 2 cycles |
| DEEP | All zones (max 3) | 3 cycles + full TRIZ principle application |

## What It Produces
- For each hot zone: thesis → antithesis → synthesis with locked constraints
- Final synthesis per zone (STRONG or flagged UNRESOLVED)
- Full audit trail: every locked constraint, every TRIZ hint applied
- Each synthesis is checked against the Concreteness Floor and Forbidden Phrases before recording (see below)

Save to `$WORKSPACE/05.7-ratchet.md` + add Ratchet syntheses to Idea DB (phase=ratchet).

## Passing Output to Phase 6

Ratchet syntheses are marked as **"pre-resolved"** when passed to the Brainwriter.
The Brainwriter builds on them — it does not re-debate them.

```bash
# Discover current schema
python scripts/idea_db.py describe <workspace>

# Add ratchet-specific column
python scripts/idea_db.py add_column <workspace> ratchet_cycle --default ""

# Ratchet syntheses are NEW ideas — use add_batch, not set_batch
# The output will print IDs: 45,46,47... — use these for set calls
python scripts/idea_db.py add_batch <workspace> ratchet-syntheses.json
# JSON format: [{"name":"...","description":"...","source_agent":"Dialectical Ratchet","chain":"Zone: [X] — Thesis → Antithesis → Synthesis","tag":"BOLD","phase":"ratchet"}]

# Set ratchet_cycle on each new synthesis using returned IDs
python scripts/idea_db.py set <workspace> <id> ratchet_cycle "2"
```

## Concreteness Floor

Before writing a synthesis to the Idea DB, verify it answers all five questions:

1. **Day 1 action** — What does the operator actually do on day 1?
2. **Money flow** — In which direction does money move, and what is the rough split (%)?
3. **Product surface** — Which specific product surface is touched (name it)?
4. **Smallest pilot** — What is the smallest runnable pilot (shippable this week)?
5. **Ownership** — Who owns this on the org chart?

Syntheses that pass: `mechanism_density=ok`. Syntheses that fail one or more: `mechanism_density=low`. Both are recorded — the Scorer downweights `low` entries.

## Forbidden Phrases

Scan every synthesis for these phrases before recording. A match triggers a mandatory rewrite:
- "becomes a VC" / "pivot to VC" / "VC fund"
- "pivots to"
- "leverage" (as a verb)
- "ecosystem play" / "platform thinking"
- "at scale"
- "network effects" (without explicit mechanism)
- "disrupts" / "reinvents"

Rewrite: replace with specific user action + product surface + money flow direction.

## Telemetry

```bash
# After recording ratchet syntheses, log:
python scripts/idea_db.py telemetry <workspace> mechanism_density_rejects <count_of_low>
```

## Anti-Patterns
- **Don't compromise in the Ratchet** — if the synthesis is just "a bit of both," run another cycle. Compromise ≠ synthesis.
- **Don't run the Ratchet on warm zones** — they don't have enough tension to produce interesting syntheses; route them to Tension Analyzer instead
- **Don't emit category labels** — "X becomes a Y" without a mechanism is a slogan. Name the check size, the flow, the surface, or flag it `mechanism_density=low`.
