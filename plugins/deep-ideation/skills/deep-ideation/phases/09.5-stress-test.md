# Phase 9.5: STRESS-TEST

Launch the Stress Tester. See `agents/stress-tester.md`.

## When to Run

After Phase 8 (SYNTHESIZE) and before Phase 10 (BRILLIANCE FILTER). Stress test results feed directly into the Brilliance Filter's evaluation — the filter checks `confidence_adjusted` to flag ideas that are both brilliant and battle-tested.

If there is no Brilliance Filter, run this before CONVERGE as the final evaluation step.

**Skipped in LITE mode.** Run in STANDARD and DEEP modes only.

## Mode Configuration

| Mode | Ideas Tested | Rounds per Idea |
|------|-------------|-----------------|
| **STANDARD** | Top 5 from Idea Menu | 2 rounds each |
| **DEEP** | Top 8 from Idea Menu | 3 rounds each |

Select the top ideas by ICE score from the Idea Menu. Include ideas from all three buckets (Quick Wins, Core Bets, Moonshots) before padding from a single bucket.

## Before Starting

```bash
# Discover current schema — check which columns exist from prior phases
python scripts/idea_db.py describe <workspace>

# Select ideas to stress-test by ICE score
# STANDARD: top 5, DEEP: top 8
python scripts/idea_db.py top <workspace> ice_score --n 5
# Or if you need bucket diversity:
python scripts/idea_db.py multi_filter <workspace> --conditions "ease>=7,confidence>=6"   # Quick Wins
python scripts/idea_db.py multi_filter <workspace> --conditions "impact>=8,confidence>=5"  # Core Bets
python scripts/idea_db.py multi_filter <workspace> --conditions "impact>=9,novelty>=8"     # Moonshots
```

Use the IDs from this output for all stress-test rounds and subsequent `set` commands.

## What It Receives

- **Idea Menu** from `$WORKSPACE/08-synthesize.md` — the top ideas with ICE scores
- **ideas.csv** from `$WORKSPACE/ideas.csv` — full metadata for each idea
- **Tension analysis** from `$WORKSPACE/07-tension.md` (if it exists) — which contradictions the team found; useful for targeting assumption attacks
- **Digger's root causes** from `$WORKSPACE/01-discover.md` — what the actual problem is; helps calibrate market fit attacks

## What It Produces

1. **Updated `ideas.csv`** — new columns added:
   - `confidence_raw` (always 5.0)
   - `confidence_adjusted` (5.0 ± round scores)
   - `stress_rounds`
   - `stress_attacks`
   - `stress_results`
   - `stress_strongest_objection`
   - `stress_modifications`

2. **Stress Test Report** saved to `$WORKSPACE/09.5-stress-test.md`

## What It Does NOT Do

- Does NOT re-score ICE — confidence here is battle-test confidence, not ICE confidence
- Does NOT generate new ideas — attacks may force modifications, but no new ideas are added
- Does NOT rank or filter — just adjusts confidence scores; the Brilliance Filter decides what to do with them

## Transition

After the Stress Tester completes, proceed to Phase 10: BRILLIANCE FILTER.

Pass the Stress Tester's notes forward. The Brilliance Filter uses `confidence_adjusted` and `stress_results` when evaluating whether an idea is both brilliant AND battle-tested. After Brilliance, CONVERGE uses all of this — stress confidence, brilliance tier, and ICE scores — to make the final decision.

## Anti-Patterns
- **Don't trust unearned confidence scores** — if an idea wasn't stress-tested, its confidence is a guess. In STANDARD and DEEP modes, `confidence_adjusted` is the number to use, not the raw ICE confidence.
- **Don't use strawman attacks in Stress Test** — the point is genuine pressure. An attack the idea can trivially deflect tells you nothing.
